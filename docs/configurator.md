# Web-конфигуратор

Web-конфигуратор — FastAPI-бэкенд + React SPA. Позволяет собрать профиль сборки через браузерный мастер из 5 шагов, запустить сборку и наблюдать за логом в реальном времени.

## Запуск сервера

```bash
python3 src/configurator/serve.py --host 0.0.0.0 --port 5173
```

После запуска откройте браузер: `http://localhost:5173`

Для доступа с другой машины замените `localhost` на IP сервера. Аутентификация отсутствует — не открывайте в публичных сетях.

---

## Мастер настройки — 5 шагов

### Шаг 1: Target

Выбор целевой платформы и параметров образа.

- **Target** — выпадающий список: `qemu-virt`, `qemu-x86`, `pi5`, `pi4`, `zero2w`
- **Image size** — размер итогового образа в МБ (пресеты: 512 МБ, 1024 МБ, 2048 МБ, или произвольное значение)
- **Boot partition size** — размер FAT boot-раздела в МБ (только для RPi target; пресет 64 МБ)

Выбор target определяет, какие опции доступны на следующих шагах (например, Wi-Fi только для `zero2w`).

---

### Шаг 2: Branding

Идентификация и доступ к системе.

- **Hostname** — hostname устройства (попадает в `/etc/hostname`)
- **Version** — строка версии образа (попадает в `/etc/netos-release`)
- **Root password** — пароль пользователя `root`
- **SSH public key** — публичный ключ для беспарольного входа по SSH (опционально)

---

### Шаг 3: Packages

Выбор пакетов, включаемых в образ.

- **110 пакетов** разбиты на **9 категорий**:
  - `core` — базовые системные утилиты
  - `networking` — сетевые инструменты (tcpdump, iperf3, ...)
  - `monitoring` — мониторинг (htop, iotop, ...)
  - `security` — безопасность (fail2ban, iptables, ...)
  - `storage` — работа с хранилищами (lvm2, mdadm, ...)
  - `ovs_kvm` — Open vSwitch, KVM-инструменты
  - `dev_debug` — отладочные инструменты (strace, gdb, ...)
  - `python` — Python-пакеты и библиотеки
  - `webui` — зависимости Web UI

- **Пресеты**:
  - **Minimal** — 6 базовых пакетов (минимальная рабочая система)
  - **Full netOS** — все ~70 пакетов по умолчанию

- **Custom BR2_PACKAGE lines** — текстовое поле для ввода произвольных строк `BR2_PACKAGE_*=y` (по одной на строку)

> **Важно**: Nervum/SDN-контроллер — не Buildroot-пакет и здесь не настраивается. Его конфигурация на шаге 4.

---

### Шаг 4: Web UI и Nervum

Настройка встраиваемого Web UI (Testum) и SDN-контроллера Nervum.

#### Web UI (Testum)

- **Source** — источник исходников:
  - `git` — клонировать из git-репозитория при сборке
  - `local` — использовать локальную директорию (путь задаётся в `source_dir`)
- **Git URL** — URL репозитория (по умолчанию `https://github.com/4stm4/testum.git`)
- **Git ref** — ветка или тег (по умолчанию `main`)
- **Local source dir** — абсолютный путь (если `source: local`)
- **Port** — порт Web UI внутри образа (по умолчанию `8080`)
- **Admin username** / **Admin password** — учётные данные администратора Web UI

#### Nervum (SDN Controller)

- **Nervum Git URL** — URL репозитория Nervum (`https://github.com/4stm4/nervum`)
- **Nervum Git ref** — ветка или тег
- **Local source dir** — локальный путь (если нужен offline-режим)

Nervum устанавливается через pip в `/opt/testum/.python` во время сборки образа. Это не Buildroot-пакет — он не добавляется в defconfig через `BR2_PACKAGE_*`.

---

### Шаг 5: Build

Запуск сборки и наблюдение за прогрессом.

- **Кнопка "Build"** — запускает `python3 src/main.py --target <target> --profile <temp_profile>` на сервере
- **Live-лог** — вывод сборки стримится через SSE (Server-Sent Events) прямо в браузер
- **Стадии прогресса**:
  1. Подготовка профиля
  2. Загрузка и компиляция ядра
  3. Загрузка и настройка Buildroot
  4. Сборка пакетов
  5. Наложение overlay, Web UI, Nervum
  6. Создание образа
- **Скачивание образа** — после успешной сборки появляется ссылка для скачивания `.img`

## Управление профилями

Конфигуратор позволяет сохранять и загружать профили через интерфейс.

- **Сохранить профиль** — сохраняет текущие настройки мастера как `.yaml` в директорию `profiles/`
- **Загрузить профиль** — импортирует профиль из файла или списка сохранённых
- **Скачать YAML** — экспортирует текущий профиль как файл для использования с `--profile`

## API endpoints

Конфигуратор предоставляет REST API (используется SPA и может использоваться напрямую):

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/api/targets` | Список доступных targets с метаданными |
| `GET` | `/api/packages` | Все пакеты с категориями и описаниями |
| `GET` | `/api/defaults` | Значения по умолчанию для профиля |
| `GET` | `/api/profiles` | Список сохранённых профилей |
| `GET` | `/api/profiles/{name}` | Получить профиль по имени |
| `POST` | `/api/profiles` | Сохранить новый профиль |
| `PUT` | `/api/profiles/{name}` | Обновить существующий профиль |
| `DELETE` | `/api/profiles/{name}` | Удалить профиль |
| `POST` | `/api/build` | Запустить сборку с переданным профилем |
| `GET` | `/api/build/log` | SSE-поток лога текущей сборки |
| `GET` | `/api/build/status` | Статус текущей сборки |
| `GET` | `/api/build/download` | Скачать собранный образ |
